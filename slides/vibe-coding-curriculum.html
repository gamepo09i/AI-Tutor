AI Builders Academy

Vibe Coding for Young Builders

A practical way to teach students how to create with AI: ask clearly, build quickly, test honestly, and improve one step at a time.

What is vibe coding?

The simple version

Vibe coding means using AI as a coding teammate. Students describe what they want, inspect what AI makes, test it, and ask for improvements.

The important part

The student is still the director. AI can suggest code, but the student chooses the goal, tests the result, and explains the changes.

Teacher note: Tell students this is not "AI does everything." It is closer to having a very fast helper who still needs clear instructions.

The class promise

  • We will build real things, not just talk about AI.
  • We will start small so every student can get a win.
  • We will use mistakes as debugging material.
  • We will explain what we made in plain English.

The prompt formula

Students can use this formula any time they get stuck.

RoleWho should AI act like?
TaskWhat should AI do?
ContextWhat details matter?
FormatHow should the answer look?

Bad prompt vs. better prompt

Too vague

"Make me a game."

Much better

"Act as a beginner-friendly JavaScript tutor. Help me make a simple clicker game in one HTML file. Add score, a timer, and comments explaining the code."

The vibe coding loop

  1. Describe the goal.
  2. Ask AI for a small first version.
  3. Run it and look carefully.
  4. Tell AI what is wrong or what to improve.
  5. Repeat until it feels good enough to share.

Lesson 1: AI teammate foundations

Goal

Students learn that AI can act as a tutor, coach, brainstorming partner, and project helper when they give it clear direction.

90-minute flow

Hook, prompt formula demo, assistant build, partner testing, revision, and exit ticket.

Activity: Have each student choose a theme: study buddy, Minecraft coach, music advisor, sports trainer, language tutor, or game idea helper.

Lesson 1 demo prompt

Act as a friendly study coach for a high school student. Help me prepare for a biology quiz about cells. Ask me one question at a time. If I get it wrong, explain it simply and give me another try.

Lesson 1: ChatGPT / menu tour

Why this prompt is weak

Bad

"Help me study."

Why it fails

No role, no topic, no format, and no interaction rule. ChatGPT has to guess what kind of help you want.

Repair prompt

Make my prompt better. Ask me for missing details first, then rewrite it using Role + Task + Context + Format.

Better prompt examples

  • Act as a biology tutor. Quiz me on cell parts one question at a time and explain mistakes simply.
  • Act as a basketball trainer. Make a 20-minute beginner dribbling practice plan.
  • Act as a game designer. Give me 3 simple HTML game ideas and rank them from easiest to hardest.

Trivia reveal: prompt quality

Question

Which prompt will usually teach better: "Help me study" or "Act as a biology tutor..."?

Answer

The tutor prompt. It includes role, topic, task, format, and an interaction rule.

Lesson 2: Codex build studio

Goal

Students open a project folder, understand files, use VS Code, ask Codex for a web page, then start a browser game.

Student output

A personal web page plus a playable Space Invaders-style starter with movement, shooting, enemies, score, and restart.

Keep setup calm. The last interactive sprint turns the folder into a tiny arcade project students can keep improving.

Lesson 2 Codex prompts

First: Create a beginner-friendly personal web page in one file named index.html. Theme: AI Builder profile Sections: name, favorite apps, project ideas, and one button Style: clean, colorful, easy to read Then: Create a beginner-friendly Space Invaders-style browser game in one file named space-invaders.html. Include player movement, shooting, falling enemies, score, lives, restart, and comments.

Lesson 2: why "make me a game" is weak

Weak prompt

"Make me a game." It does not name the game type, file, controls, difficulty, required features, or how much code is okay.

Better prompt

Create a beginner-friendly Space Invaders-style game in one HTML file. Include player movement, shooting, falling enemies, score, lives, restart, and comments.

Fix prompt: Rewrite this as a Codex prompt with exact file name, controls, features, and a request to explain the code.

Lesson 2 final interactive: Space Invaders basics

Core loop

Move the ship, shoot bullets, hit enemies, update score.

Debug loop

Run it, describe what broke, ask Codex for one focused fix.

Homework

Add a power-up, enemy type, level, sound, or visual polish.

Teacher note: Games are great because bugs become visible. If movement, scoring, collisions, or reset fails, students can describe the problem clearly.

Lesson 2 game improvement prompts

  • The bullets do not hit enemies. Find the collision bug and change only the needed code.
  • Add a start screen and game over screen.
  • Make enemies faster after every 10 points.
  • Add a power-up that gives double shots for 8 seconds.
  • Explain the collision code like I am new to JavaScript.

Trivia reveal: debugging

Question

Which is better: "It is broken" or "Bullets appear, but they do not disappear when they hit enemies"?

Answer

The second. It tells Codex what happened, what should happen, and where to look.

Lesson 3: Agents, research, and browser safety

Chatbot

Answers questions inside the conversation.

Agent

Plans steps, uses tools, checks results, and reports back.

Simple phrase: An agent is AI plus a job plus tools.

Lesson 3: prompt repair for research

Weak prompt

"Find videos about coding." It does not say beginner or advanced, how many videos, what to compare, or what to verify.

Better prompt

Compare 3 beginner-friendly YouTube videos about making a simple JavaScript game. Give topic, strongest part, weakness, and which one to watch first.

Fix prompt: Turn this into a safe research-agent prompt. Add source checking, beginner criteria, and a table format.

Lesson 3: YouTube research workflow

  1. Pick a topic students care about.
  2. Find three useful videos.
  3. Summarize the main ideas.
  4. Compare which video is best for beginners.
  5. Mark what a human should verify before trusting the report.

Lesson 3: Browser Use and Computer Use

What students learn

Some AI systems can operate a browser or computer-like interface, but they need safe boundaries and clear tasks.

Safe classroom examples

Compare information, collect public facts, fill a practice form, or summarize a webpage.

Lesson 3 safety rules

  • Do not enter private passwords during demos.
  • Do not let AI buy anything or submit real forms without permission.
  • Use practice websites or public information.
  • Watch what the agent is doing and be ready to stop it.

Lesson 3 examples students can copy

  • What facts should I verify before trusting this video summary?
  • Design a safe 5-step public-information browser task with one human approval checkpoint.
  • Make a table with video title, skill level, strongest part, weakness, and recommendation.

Trivia reveal: safe agent tasks

Question

Which is safer: buying something online, logging into email, or comparing public tutorials?

Answer

Comparing public tutorials. It avoids private accounts, purchases, and irreversible actions.

Lesson 4: prompt repair for presentations

Weak prompt

"Make my presentation good." It does not explain the project, audience, time limit, or what must be included.

Better prompt

Help me write a 2-minute demo script for my Space Invaders project. Audience: classmates. Include what I built, one bug I fixed, one prompt that helped, and one feature I would add next.

Trivia reveal: final demos

Question

What makes a final demo strong?

Answer

One clear working moment, one problem it solves, one thing that broke, and one improvement idea.

Student presentation format

  1. What did you build?
  2. Who is it for?
  3. What prompt helped the most?
  4. What broke, and how did you fix it?
  5. What would you add next?

How to teach when you are new too

You do not need to be the person with every answer. Be the person who models how to ask, test, debug, and keep going.

Line you can use: "I do not know yet. Let's ask clearly, test the answer, and see what changes."

Your first repo workflow

  1. Put the useful files in the folder.
  2. Commit the files as one named update.
  3. Push to GitHub.
  4. Tell people to download or pull the newest version.
Closing message

Build small. Test often. Explain what changed.

That is the heart of vibe coding, and it is enough to get students making real projects quickly.
